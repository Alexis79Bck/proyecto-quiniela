# Plan: Tests de Autenticación (Sanctum + Fortify)

## Objetivo
Crear un set básico y mínimo de tests para verificar la funcionalidad del sistema de autenticación, combinando con el sistema de notificaciones ya existente.

## Tests a Implementar

### 1. Authentication Tests (`tests/Feature/Auth/AuthenticationTest.php`)

| Test | Descripción |
|------|-------------|
| `test_user_can_register` | Verifica que un usuario puede registrarse correctamente y recibe un token |
| `test_user_can_login` | Verifica que un usuario puede iniciar sesión y recibe un token |
| `test_login_fails_with_invalid_credentials` | Verifica que credenciales inválidas retornan error |
| `test_user_can_logout` | Verifica que el logout elimina el token correctamente |
| `test_unauthenticated_user_cannot_access_protected_routes` | Verifica que rutas protegidas requieren autenticación |
| `test_authenticated_user_can_get_own_info` | Verifica que el endpoint /api/me retorna datos del usuario autenticado |

### 2. Combined Auth + Notifications Tests (`tests/Feature/Auth/AuthenticatedNotificationsTest.php`)

| Test | Descripción |
|------|-------------|
| `test_authenticated_user_can_list_notifications` | Usuario autenticado puede listar sus notificaciones |
| `test_authenticated_user_can_mark_notification_as_read` | Usuario puede marcar notificación como leída |
| `test_user_cannot_access_other_user_notifications` | Usuario no puede acceder a notificaciones de otro usuario |

## Estructura de Archivos a Crear

```
tests/Feature/Auth/
├── AuthenticationTest.php
└── AuthenticatedNotificationsTest.php
```

## Consideraciones de Implementación

1. **Base**: Extender de `Tests\TestCase` con trait `RefreshDatabase`
2. **Autenticación**: Usar `actingAs($user, 'sanctum')` para simular usuario autenticado
3. **Usuario de prueba**: Usar `User::factory()->create()` para crear usuarios
4. **Assertions**: Usar `assertJsonStructure`, `assertStatus`, `assertDatabaseHas`
5. **Rutas API**: Testear usando `getJson()`, `postJson()`, `deleteJson()`
